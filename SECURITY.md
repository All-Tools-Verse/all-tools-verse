# Security policy

## Reporting a vulnerability

Do not open a public GitHub issue for a vulnerability or a report containing sensitive information.

Email `contact@alltoolsverse.com` with:

- the affected All Tools Verse URL;
- a clear description of the issue and potential impact;
- reproducible steps or a minimal proof of concept;
- any suggested mitigation;
- a safe way to contact you.

Please avoid accessing, modifying or retaining data that does not belong to you. Allow reasonable time for investigation before public disclosure.

## Repository scope

This repository contains a public directory and maintenance scripts, not the production application source code. Broken links, wrong titles and category mismatches can be reported with the normal issue forms; security-sensitive findings should use email.
